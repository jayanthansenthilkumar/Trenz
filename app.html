App Dev Pro Challenge

App Dev Pro Challenge 2025

April 30, 2025
09:00 AM - 4:00 PM
Individual/Team (max 2 members)

Rules & Guidelines

  • Name of the Event: APP A THON
  • Event Overview: Participants will build a functional web application integrated with a database within the given time limit. The focus is on real-time problem-solving, effective database usage, and efficient coding practices.
  • Participation Format:
    • Team participation only — Maximum 2 members per team.
    • Each team can submit only one project.
  • Event Structure:

    Development Phase:

    • Teams can develop the web app based on their own topic.
    • 3 hours will be provided to build a complete web app with database integration.

    Demonstration Phase:

    • Each team will briefly present their app to the judges (5–7 minutes).
  • General Rules:
    • All apps must be developed from scratch during the event duration.
    • Design is not important, marks will be provided based on the functionalities of the web app.
    • Database integration is mandatory (e.g., MySQL, MongoDB, Firebase, PostgreSQL, etc.).
    • Any programming language, framework, or database technology can be used.
    • Use of online resources (documentation, libraries) is allowed. Pre-built projects/templates are not allowed.
    • Projects must be submitted within 3 hours. Late submissions will not be evaluated.
    • Teams must demonstrate the functionality of their web app live in front of the judges.
    • Respect towards organizers, judges, and fellow participants is expected at all times.
  • Technical Requirements:
    • The app must run on a standard browser (e.g., Chrome, Firefox).
    • A working database backend must be demonstrated (insert, retrieve, update, delete operations).
    • Teams must submit their source code at the end of the event.
    • If hosting is not possible within the time, a local server demonstration is acceptable.
  • Judging Criteria:
    • Functionality and Working Model - 40%
    • Database Design and Usage - 35%
    • Innovation and Creativity - 15%
    • Code Quality and Structure - 10%
  • Bonus Points:
    • Extra points for creative database structures (e.g., relational integrity, optimized queries).
    • Bonus for real-world applicability and scalability ideas.
  • Important Notes:
    • Maintain frequent local saves/backups — no excuses for data loss.
    • Focus on creating, innovating, and learning.
    • Stay calm, collaborate effectively, and code smart!
    • Judges' decisions will be final and binding.
